## 🎯 The Golden Story Formula (use this every time)

```
MESS / STRESS → EASY BOOKING → TRUSTED CLEANER → VISIBLE TRANSFORMATION → RELIEF / CONFIDENCE → CTA
```

This is the strongest recurring ad engine for PureTask. Build every video around this arc.

### Script Structure (6 beats — always in this order)
1. **Hook** — Grab attention in the first 2 seconds. Specific, not generic.
2. **Problem** — Name the stress, mess, or inconvenience. Make it real.
3. **Solution** — Introduce PureTask clearly and simply.
4. **Benefits** — 2–4 concrete benefits. Show, don't just say.
5. **Trust Layer** — Reliability, ease, proof. Make it feel safe.
6. **CTA** — One clear action. Direct. Simple.

---

## ✍️ Scripting Rules (non-negotiable)

- Write in **spoken language** — natural, simple, direct, conversational
- No stiff corporate wording. No abstract claims. No jargon.
- Every sentence should sound like a real human talking, not reading
- One audience per video. One main message per video. One emotional transformation.
- Never cram multiple features into one script — pick the strongest one
- Scripts must feel: **"this looks legit, modern, and easy to use"**
- Always end with a clear CTA that names the action + URL

**Scripting formula for customers:**
> Start with overwhelm or mess → show how easy PureTask makes booking → show a cleaner transforming the home → end with relief and a CTA

**Scripting formula for cleaners:**
> Emphasize flexibility → choosing jobs that fit their life → building reputation → growing income → CTA to apply

### Color System
```
Primary blue:     #0099FF
Background:       #FFFFFF (white) or #F5F7FA (light gray)
Dark text:        #1A1A2E

URL:              https://www.puretask.co — must appear visually in EVERY video
```

### Visual Tone (always)
- Clean and modern
- Bright interiors with soft premium lighting
- Organized compositions — nothing cluttered
- Realistic but polished
- Friendly human energy
- Smooth transitions

### Preferred Scenes
- Tidy modern homes — kitchens, bathrooms, living rooms
- Happy customers booking on their phone
- Cleaners arriving, cleaning, organizing — confident and professional
- Sparkling clean finished rooms
- Clean UI overlays, booking steps, confirmation moments
- Lifestyle moments showing relief and satisfaction

### NEVER Use
- Dark moody visuals or dramatic shadows
- Chaotic or aggressive editing
- Overly futuristic tech aesthetics
- Cheesy corporate stock photo energy
- Cluttered text-heavy scenes
- Exaggerated or unrealistic claims
- Generic "stock photo couple smiling at laptop" energy

### Before/After Contrast (priority visual technique)
PureTask videos work best when they visually contrast:
- Mess → Clean
- Stress → Relief
- Confusion → Clarity
- Wasted time → Convenience
- Inconsistent work → Flexible opportunities

## 🎭 Avatar Strategy

### USE an Avatar When:
- Making explainers or spokesperson videos
- Making founder-style or direct communication promos
- Making customer-facing content that needs a human voice and face
- Making recruitment messages

- Making lifestyle montage ads
- Making cinematic social ads
- Making before/after cleaning transformation videos

## ✅ Quality Checklist (Every Video — ALL Must Pass Before Generating)

**Format**
- [ ] Duration matches platform target
- [ ] Aspect ratio matches platform
- [ ] Audience defined — one specific group
- [ ] Objective defined — one specific goal
- [ ] One main message — not multiple features stacked

**Script**
- [ ] Full spoken script written (not just a topic summary)
- [ ] Sounds natural and conversational — not corporate
- [ ] Hook grabs in first 2 seconds — specific, not generic
- [ ] Story arc: problem → solution → benefit → trust → CTA
- [ ] CTA is specific ("Book at https://www.puretask.co" not "learn more")
- [ ] Real brand stats used where relevant (4.9★, 10K+, 98%, 2,400+, 6hrs, 80–85%)
- [ ] No hype words, no fake urgency, no financial guarantees

**Visuals**
- [ ] `https://www.puretask.co` appears visually in video
- [ ] PureTask blue (#0099FF) used in visuals/overlays
- [ ] Before/after contrast shown (mess → clean / stress → relief)
- [ ] No dark moody shots, no cheesy stock, no clutter
- [ ] Media types assigned: stock / motion graphics / AI visuals

**Production**
- [ ] Avatar or VO-only — explicitly stated
- [ ] Correct avatar ID included
- [ ] Correct voice ID included
- [ ] Subtitles enabled
- [ ] Intro + outro cards included
- [ ] Scene-by-scene breakdown written
- [ ] Music direction specified
- [ ] Tied to one of the 6 content pillars

**Scoring (post-generation)**
- Avg ≥ 7.5 → Auto-approved, post to platform
- Avg 5.0–7.4 → Regenerate with revised prompt
- Avg < 5.0 → Reject, log reason, rebuild from scratch

**Video-specific score penalties:**
- No URL visible in video → -2 Clarity
- No subtitles → -2 Conversion
- Hook > 3 seconds → -2 Relatability
- No CTA in final scene → -3 Conversion
- Wrong aspect ratio → Auto-reject
- Cheesy stock / dark moody vibe → -2 Relatability

## 🔁 API Generation Workflow

### Step 1 — Submit Generation Request
```json
POST https://api.heygen.com/v2/video/generate
X-Api-Key: $HEYGEN_API_KEY

{
  "video_inputs": [
    {
      "character": {
        "type": "avatar",
        "avatar_id": "Abigail_expressive_2024112501",
        "avatar_style": "normal"
      },
      "voice": {
        "type": "text",
        "voice_id": "PXNEIUJiwgmsriDe9m6P",
        "input_text": "[FULL SCRIPT HERE]",
        "speed": 1.0
      },
      "background": {
        "type": "color",
        "value": "#FFFFFF"
      }
    }
  ],
  "aspect_ratio": "9:16",
  "test": false
}
```

### Step 2 — Poll Status (every 30 sec)
```
GET https://api.heygen.com/v2/video/status/{video_id}
X-Api-Key: $HEYGEN_API_KEY
```
Statuses: `pending` → `processing` → `completed` | `failed`

### Step 3 — On Completion
1. Extract `video_url` from response
2. Upload to CDN via `upload_file` tool
3. Store CDN URL in `ContentDraft.video_prompt` field
4. Update `ContentDraft.status` → "Approved"
5. Post to platform via Ayrshare using video URL
6. Update `ContentDraft.posted_platforms` → "Posted"
7. Log to `/videos/finished-videos.md` on GitHub
